## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

```
1. アイドリング画面を表示する。
2. ユーザーの操作を待機する。
3. 操作に応じて以下の処理を実行する。
   - 商品ページを表示する。
     1. 商品一覧を表示する。
     2. 商品を選択し、選択内容を確認する。
     3. 確認後、金額投入画面に進む。
   - 金額投入画面を表示する。
     1. 投入金額を入力する。
     2. ポイント利用の有無を確認する。
     3. 購入確定後、お釣りを計算して排出する。
   - キャンセル操作が行われた場合。
     1. 投入金額を返金する。
     2. 操作ログを記録する。
4. 処理終了後、アイドリング画面に戻る。
```

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: 商品の選択（`select_product`）
```
1. ボタンを押して商品の選択を行う。
2. 選択した商品を強調表示する。
3. 押した商品IDを取得する。
4. 購入ボタンを有効化する。
```

#### 機能2: 投入金処理（`input_money`）
```
1. 画面に「投入金を入力してください」と表示する。
2. ユーザからの入力を受け取る（`scanf`を使用）。
3. 入力値が非負整数（`unsigned int`型）であるかを確認する。
   - 非負整数の場合: 変数に格納する。
   - それ以外の場合: エラー「正しく入力してください」を表示する。
```

#### 機能3: 購入確定処理（`confirm_purchase`）

```
1. 選択された商品のIDと数量を取得する。
2. 在庫数を確認する。
   - 在庫が足りない場合、エラーを表示して処理を中断。
3. 合計金額を計算する。
4. 会員IDが入力されている場合、ポイント利用の確認を行う。
   - ポイント利用額を差し引いた金額を計算。
5. 投入金額と合計金額を比較する。
   - 投入金額が不足している場合、エラーを表示して処理を中断。
6. 購入を確定し、在庫数を更新する。
7. 売上データを保存する（`save_sales_data`を呼び出し）。
8. 操作ログを記録する（`save_log`を呼び出し）。
```

#### 機能4: お釣りの計算・排出処理（`calculate_change`）

```
1. 投入金額から合計金額を差し引いてお釣りを計算する。
2. お釣りを排出する。
3. 操作ログにお釣り排出の記録を保存する（`save_log`を呼び出し）。
```

#### 機能5: キャンセル時の返金処理（`cancel_purchase`）

```
1. 投入金額を確認する。
2. 投入金額を全額返金する。
3. 操作ログにキャンセル処理の記録を保存する（`save_log`を呼び出し）。
```

#### 機能6: 会員認証・ポイント管理処理（`member_login`）

```
1. 会員IDを入力させる。
2. 入力されたIDを会員データから検索する。
   - 該当する会員がいない場合、エラーを表示して処理を中断。
3. 会員情報（ポイント残高など）を表示する。
4. ポイント利用の有無を確認する。
   - 利用する場合、利用ポイントを入力させる。
   - ポイント残高を更新する（`update_points`を呼び出し）。
```

#### 機能7: 商品入れ替え処理（`replace_products`）
```
1. 管理者ログインを確認する。
2. 入れ替え対象の商品IDを入力させる。
3. 新しい商品情報（商品名、単価、在庫数、温冷区分）を入力させる。
4. 入力内容を確認し、データを更新する。
5. 商品入れ替え履歴を保存する（`save_log`を呼び出し）。
```

#### 機能8: 故障・異常検知処理（`detect_fault`）
```
1. センサーやシステムの状態を定期的に監視する。
2. 異常を検知した場合、異常内容を記録する。
3. 管理者に通知を送信する。
4. 故障・異常情報を保存する（`save_log`を呼び出し）。
```

#### 機能9: 管理者用メニュー表示・操作（`admin_menu`）
```
1. 管理者ログインを確認する。
2. 管理者用メニューを表示する。
   - 商品入れ替え
   - システム状態確認
   - ログ閲覧
3. 選択された操作を実行する。
```

### 4.3 エラーハンドリング（想定される異常系）

> ユーザーが「想定外の操作」をした場合にどうするか検討してください。

| 異常系シナリオ | 対処方針 |
| :--- | :--- |
| 数字以外の文字や不正な値が入力された場合 | エラーメッセージを表示し、正しい入力を再度促す |
| 存在しない商品IDや選択肢が選ばれた場合 | エラーメッセージを表示し、商品選択画面に戻す |
| 購入数量が在庫数を超えている場合 | エラーメッセージを表示し、数量の再入力を促す |
| 投入金額が不足している場合 | エラーメッセージを表示し、追加投入またはキャンセルを促す |
| 会員IDが未入力・誤入力・未登録の場合 | エラーメッセージを表示し、再入力またはスキップを促す |
| ポイント利用額が残高を超えている場合 | エラーメッセージを表示し、利用額の再入力を促す |
| 操作途中でキャンセルされた場合 | 直前の状態に戻し、必要に応じて返金・ログ記録を行う |
| 管理者認証に失敗した場合 | エラーメッセージを表示し、再認証を促す |
| 商品入れ替えや設定値が不正な場合 | エラーメッセージを表示し、再入力を促す |
| センサー異常やシステムエラーが発生した場合 | 異常内容を記録し、管理者に通知・画面にエラー表示 |
| 返金処理が正常に行えなかった場合 | エラーメッセージを表示し、管理者に通知する |
| ログ保存に失敗した場合 | エラーメッセージを表示し、再試行または管理者に通知する |
| ログファイルが破損・読み込めない場合 | エラーメッセージを表示し、バックアップからの復旧や管理者に通知する |
| ログ閲覧時に権限がない場合 | エラーメッセージを表示し、管理者認証を促す |

## 6. 関数シグネチャー案（基本設計書準拠）

```c
// --- データ構造体定義例 ---
// 商品情報をまとめる構造体
typedef struct {
    int product_id;      // 商品ID
    char name[64];      // 商品名
    unsigned int price; // 単価
    int stock;          // 在庫数
    char type[8];       // 温冷区分（"冷"や"温"など）
} ProductInfo;

// 売上データをまとめる構造体
typedef struct {
    int product_id;         // 商品ID
    int quantity;           // 購入数量
    int member_id;          // 会員ID
    unsigned int use_point; // 利用ポイント
    unsigned int change;    // お釣り
} SalesData;

// 購入処理に必要な情報をまとめる構造体
typedef struct {
    int product_id;         // 商品ID
    int quantity;           // 購入数量
    unsigned int money;     // 投入金額
    int member_id;          // 会員ID
    unsigned int use_point; // 利用ポイント
} PurchaseInfo;

// --- 関数シグネチャー案（構造体利用） ---

// システムの初期化（データ読込・初期設定）
// 戻り値: 0=成功, 1=データ読込失敗, 2=初期設定エラー
int init_system(void);

// アイドリング画面の表示
void show_idle_screen(void);

// 商品一覧画面の表示
void show_product_page(void);

// 商品選択処理（商品ID入力・選択）
// 戻り値: 選択された商品ID（-1: キャンセル, -2: 入力エラー）
int select_product(void);

// 商品選択内容の確認画面表示
// 引数: product_id - 商品ID, quantity - 購入数量
// 戻り値: 0=確認OK, 1=数量エラー, 2=キャンセル
int confirm_selection(int product_id, int quantity);

// 金額投入処理（投入金額の受付・表示）
// 戻り値: 投入された金額（0: キャンセル, -1: 入力エラー）
int input_money(void);

// 購入確定処理（合計金額計算・在庫確認）
// 引数: 購入情報構造体（商品ID、数量、投入金額、会員ID、利用ポイント）
// 戻り値: 0=成功, 1=在庫不足, 2=金額不足, 3=その他エラー
int confirm_purchase(PurchaseInfo purchase);

// お釣りの計算・排出処理
// 引数: money - 投入金額, total_price - 合計金額
// 戻り値: お釣り金額
unsigned int calculate_change(unsigned int money, unsigned int total_price);

// キャンセル時の返金処理
// 引数: money - 投入金額
// 戻り値: 0=返金成功, 1=返金失敗
int cancel_purchase(unsigned int money);

// 在庫数の更新処理
// 引数: product_id - 商品ID, quantity - 増減数（マイナスで減少）
// 戻り値: 0=成功, 1=在庫不足, 2=更新失敗
int update_inventory(int product_id, int quantity);

// 売上データのCSV保存処理（3回まで自動再試行）
// 引数: 売上データ構造体
// 戻り値: 0=成功, 1=保存失敗
int save_sales_data(SalesData sales);

// 操作ログのCSV保存処理（3回まで自動再試行）
// 引数: operation - 操作内容, detail - 詳細, error - エラー内容
// 戻り値: 0=成功, 1=保存失敗
int save_log(const char* operation, const char* detail, const char* error);

// 会員認証・ポイント管理処理
// 引数: member_id - 入力された会員ID
// 戻り値: 0=認証成功, 1=認証失敗
int member_login(int member_id);

// ポイント付与・利用処理
// 引数: member_id - 会員ID, add_point - 付与ポイント, use_point - 利用ポイント
// 戻り値: 0=成功, 1=残高不足, 2=更新失敗
int update_points(int member_id, int add_point, int use_point);

// 商品入れ替え処理（管理者用）
// 引数: 商品情報構造体
// 戻り値: 0=成功, 1=入力エラー, 2=更新失敗
int replace_products(ProductInfo product);

// 故障・異常検知処理
// 戻り値: 0=異常なし, 1=異常検知
int detect_fault(void);

// 管理者用メニュー表示・操作
// 戻り値: 0=正常終了, 1=認証失敗, 2=操作エラー
int admin_menu(void);
```
