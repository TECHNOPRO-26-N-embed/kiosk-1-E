## 2. 要求仕様書（顧客ヒアリング結果）



### 2.1 顧客の要望（ヒアリングメモ）

> ここに、顧客（講師）へのヒアリングで得た要望を箇条書きで記録してください。

* 在庫は1種類あたり50個
* 同時に扱う商品は50種まで
* セキュリティ面は出来るだけ考慮、絶対に止まらないように
* 売上データの他に故障予知などできればほしいです
* 画面はテキストメニューでかっこよく
* 商品の選択・数量入力・合計計算ができるように
* 操作ログ(売上データ等)をCSVファイルに残したい
* 複数のお客さんが続けて使えるように
* 季節毎に製品を入れ替え
* 在庫管理はあれば嬉しい
* ポイントカードや会員機能はあれば嬉しい
* 温冷の区別はデータとして持つなら嬉しいが表示だけでOK

### 2.2 機能要件

| 機能ID | 機能名 | 概要 |
| :---: | :--- | :--- |
| F01 | 商品の選択 | 選ばれた商品の強調 |
| F02 |  投入された金額の読み取り | 投入された金額の表示 |
| F03 | 在庫計算 | 売り切れの場合は「うりきれ」表示 |
| F04 |お釣り計算 | 商品額と投入額の差分を輩出 |
| F05 | 商品の入れ替え記録 | 商品入れ替え時に記録 |
| F06 | 購入履歴保存 | 売上データをCSVファイルに記録 |
| F07 | キャッシュ条件管理 | 容量状況をCSVファイルに記録 |
| F08 | 返金 | 商品選択がされなかった場合投入された金額を返金 |
