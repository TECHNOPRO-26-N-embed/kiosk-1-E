## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
1. メインメニューを表示する
2. ユーザーの入力を受け取る
3. 入力に応じて分岐する
   - 1の場合: 商品選択画面 の表示
   - 2の場合: 金額投入
   - 9の場合: プログラムを終了する
4. メインメニューに戻る（1へ）
```

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: （商品の選択）

```
・ボタンを押して商品の選択
・選択した商品を強調表示
・押した商品IDを取得する
・購入ボタンを有効化
```

#### 機能2: （投入金）
```
画面に投入金を入力してくださいと表示
ユーザからの入力を受け取る(scanf)
→非負整数unsigned int型なら→変数に格納
→それ以外なら→エラー「正しく入力してください」
```

#### 機能3: 在庫計算
```
1.1未満であれば「売り切」と表示し、ボタンをfalseを設定変更します。
2.購入する時、ボタンを事前判定し、tureかfalseかをボタンの有効化が判定します。
　tureであれば在庫1以上で在庫数を-1とします。
```

#### 機能4: （お釣り計算）

```
入力データ：
・投入金額
・商品金額
出力データ
・各硬貨の排出枚数
扱う硬貨の種類
・500, 100, 50, 10円硬貨 

１, お釣り総額の計算
お釣り総額 = 投入金額 - 商品金額 を計算する

２, 大きい硬貨から順番に確認
・硬貨のリスト(500 → 100 → 50 → 10円)を上から順番に確認する

３, 排出枚数の算出
・硬貨の排出枚数 = お釣り総額 ÷ 硬貨の金額
・お釣り総額をその硬貨で払いきれなかった余りに更新する
 - 新しいお釣り総額 = 現在のお釣り総額 ÷ 硬貨の金額 の余り(モジュロ演算)

４, 終了判定
・お釣り総額が０になった。あるいはすべての硬貨(10円まで)の処理が終わったらループ解除

５, 結果の出力
・算出した各硬貨の枚数を払い出し口に排出する

```

#### 機能5: 商品の入れ替え記録
1.csvファイル読み込み→ 2-1.[商品入れ替え]→ 3-1.商品ID, 単価, 在庫数量の商品情報をcsvファイルに書き込み
#### 機能6: 商品の購入履歴・売上計算
1.csvファイル読み込み→ 2-2.[商品購入]→ 3-2.商品ID, 単価, 購入日時, 在庫数量の購入履歴をcsvファイルに追記→ 4-2.購入履歴を基に、決まった時間に売上計算(購入した商品の単価×商品数)を行う　

#### 機能7: （取り消し処理）
```
キャンセルボタンが押されたら「ありがとうございました(返却○○円)」と表示
メインメニューに戻る
```
