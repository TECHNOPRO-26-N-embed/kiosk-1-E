### 3.2 データの洗い出し

> このシステムが**覚えておくべき情報**を洗い出してください。
> まずは「何の情報が必要か」を自然言語で書き出せばOKです。
> （C言語での型や構造体の定義は、明日の詳細設計で行います）

| 情報の名前 | 内容・説明 | 例 |
| :--- | :--- | :--- |
| 商品ID | 商品を一意に識別する番号 | 1(コーラ) |
| 商品名 | 商品の名称 | コーラ、サイダーなど |
| 単価 | 商品1つあたりの価格 | 120円 |
| 在庫数 | 各商品の残り個数 | コーラ残り10本 |
| 温冷区分 | 商品が温かいか冷たいか | 冷、温 |
| 投入金額 | 現在投入されている金額 | 500円投入 |
| お釣り | 返却する金額 | 80円返却 |
| 購入数量 | 購入した商品の個数 | 2本 |
| 売上データ | 購入日時・商品・数量・金額 | 2026/05/19,コーラ,2,240円 |
| 操作ログ | 操作やエラーの記録 | 商品選択、エラー発生など |
| 会員ID | 会員を識別する番号 | 1001 |
| ポイント | 会員のポイント残高 | 30ポイント |
| 商品入れ替え履歴 | 季節ごとの商品変更記録 | 2026春:新商品追加 |
| 故障・異常情報 | 故障予知や異常検知の記録 | 異常発生日時、内容 |
| 管理者情報 | システム管理者の識別や設定 | 管理者ID、パスワード |

### 3.3 CSV保存形式

> CSVファイルにどのような形式でデータを保存するか定義してください。
id, 単価, 購入日時, 在庫数量  

#### 売上データ（購入履歴）
```
購入日時,商品ID,商品名,単価,購入数量,合計金額,会員ID,ポイント利用,お釣り
例: 2026/05/19 10:00,1,コーラ,120,2,240,1001,10,80
```

#### 在庫データ
```
商品ID,商品名,在庫数,温冷区分
例: 1,コーラ,10,冷
```

#### 操作ログ
```
日時,操作内容,詳細,エラー内容
例: 2026/05/19 10:01,商品選択,コーラ,なし
```

#### 会員データ
```
会員ID,会員名,ポイント残高,最終利用日時
例: 1001,山田太郎,30,2026/05/19 10:00
```

#### 商品入れ替え履歴
```
日時,商品ID,商品名,操作内容
例: 2026/03/01,5,春限定ドリンク,追加
```

#### 故障・異常情報
```
日時,内容,対応状況
例: 2026/05/19 11:00,在庫センサー異常,未対応
```
